# Visual Template Profiles

Use this reference when creating `designed-resume.html`, `designed-resume.pdf`, `designed-resume.docx`, Figma handoff, Canva handoff, or visual design review artifacts.

## Template Selection Rules

Always keep the ATS application resume separate from the visual resume. The visual template may use stronger hierarchy, profile picture slots, accent colors, and richer layout, but the ATS resume remains text-first and parse-safe.

Pick one named visual profile and record the decision in `visual-template-selector.md`.

## Profiles

### Modern Technical

Best for senior engineers, staff engineers, platform engineers, cloud engineers, AI/automation engineers, and technical leads.

- Layout: strong header, two-column skills band, experience-first body.
- Visual emphasis: technical themes, impact metrics, architecture scope, open-source work.
- Color: restrained neutral base with one professional accent.
- Avoid: decorative graphics, dense sidebars, icon-heavy skills.

### Executive Compact

Best for engineering managers, principal engineers, director-track profiles, and leadership-heavy candidates.

- Layout: compact summary, leadership scope, impact themes, selected achievements, experience timeline.
- Visual emphasis: business impact, org scope, strategy, delivery, stakeholder influence.
- Color: conservative, high contrast, print-friendly.
- Avoid: over-designed visuals that reduce credibility.

### Open Source Builder

Best when public GitHub, maintainership, community work, public talks, or public technical writing strengthen the profile.

- Layout: experience plus public work panel.
- Visual emphasis: repositories, releases, community impact, adoption, technical credibility.
- Color: clean technical palette with clear link styling.
- Avoid: implying employer-owned private code is public.

### Enterprise Platform

Best for enterprise, internal platform, GitHub Enterprise, Confluence/Jira-heavy evidence, and confidential work that must be sanitized.

- Layout: sanitized project descriptions, outcome-led bullets, architecture and reliability themes.
- Visual emphasis: scale, reliability, migration, security, compliance, developer productivity.
- Color: quiet professional palette, strong section hierarchy.
- Avoid: private project names, customer names, repository names, private URLs, or exact sensitive metrics without approval.

### AI Automation

Best for profiles centered on AI tooling, automation, developer productivity, LLM workflows, data platforms, or applied AI systems.

- Layout: summary with AI/platform focus, project case studies, metrics, skills grouped by AI, platform, delivery, and operations.
- Visual emphasis: measurable productivity, quality, reliability, adoption, and governance.
- Color: modern but not trend-driven.
- Avoid: vague AI claims without evidence.

## Visual Quality Gates

Record checks in `visual-design-review.md`:

- First-page role fit is obvious within a quick skim.
- Name, target role, summary, strongest impact, and top skills appear early.
- Text does not overlap, clip, or shrink below readable size.
- Dates, company names, and skills remain parseable by a human.
- Profile photo is omitted unless user-provided and approved.
- Print/PDF output has acceptable page breaks.
- Visual artifact does not replace `ats-resume.docx` for job portals.
- Canva/Figma third-party upload status is explicit.

## Profile Picture Use

Use a profile picture only in visual artifacts and only when the user provides or approves the image. Do not include profile pictures in ATS resumes. Do not upload images to Canva, Figma, or any third-party service without explicit user approval.
