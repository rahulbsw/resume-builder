# Career and Interview Coaching Workflow

Use this reference when the user asks for a career coach, interview coach, professional positioning, full work timeline, or detailed resume for interview preparation.

## Coaching Inputs

Start from evidence-backed artifacts:

- `source-inventory.md` and `tool-usage-log.md` for source coverage.
- `evidence.md` for claims, confidence, source contexts, and disclosure.
- `detailed-resume.md` for the full timeline, roles, responsibilities, projects, impact, and metrics.
- `resume-scorecard.md` for ATS, recruiter, keyword, and claim-risk findings.
- `job-match-report.md`, `gap-analysis.md`, or job postings when coaching is role-specific.

Do not invent career strategy claims. If the evidence does not support a seniority, domain, metric, or leadership scope, ask the user for confirmation or keep it in the gap list.

## Career Master Run

Create `career-master-run.md` when the user wants a comprehensive preparation package or is rebuilding career history across LinkedIn, local docs, Confluence, Jira, and GitHub.

Include:

- Career positioning thesis.
- Target role families, seniority, and geographic/work-model preferences.
- Evidence coverage map by role and source.
- Strength themes supported by evidence.
- Weak or missing areas that need user answers.
- Resume strategy: ATS version, visual version, detailed interview version, LinkedIn updates, and job-specific variants.
- Interview strategy: strongest stories, technical depth, behavioral bank, and risks to rehearse.
- Job-search strategy: match categories, keywords, stretch roles, and roles to avoid.

## Missing Information Loop

Create `missing-information-loop.md` when key facts are missing. Group questions by priority:

- High: dates, title, ownership, exact contribution, metrics, public disclosure, and sensitive names.
- Medium: scale, team size, architecture details, collaborators, tradeoffs, and incident context.
- Low: wording preference, optional certifications, public links, and design preferences.

Ask concise questions. Avoid sending a long questionnaire unless the user explicitly asks. Preserve unanswered questions in the run folder so future sessions can continue.

## Career Positioning Coach

Create `career-positioning-coach.md` after the detailed resume or scorecard when the user wants stronger positioning.

The coach output should include:

- Best-fit role families with reasoning.
- Positioning statement options for resume, LinkedIn headline, recruiter screen, and interview introduction.
- Seniority evidence: scope, leadership, architecture, delivery, and cross-functional influence.
- Differentiators: enterprise work, open-source work, domain expertise, cloud/platform depth, AI/automation, reliability, security, or developer productivity.
- Risks: unsupported keywords, weak metrics, unclear ownership, overly internal wording, or mismatched seniority.
- Recommended next moves: resume edits, LinkedIn changes, proof to gather, and job-search filters.

## Interview Coaching Loop

Create `interview-readiness-scorecard.md` when the user asks for interview preparation or when a target job is selected.

Score readiness across:

- Resume walk-through.
- Recruiter screen narrative.
- Key project deep dives.
- Technical stack explanations.
- System design preparation.
- Behavioral stories.
- Job-specific requirement mapping.
- Redaction and disclosure safety.

Use evidence-backed answer notes. Mark gaps clearly and ask for missing metrics, dates, or ownership details.

## Coaching Guardrails

- Keep coaching practical and specific to the user's target roles.
- Separate application resume advice from interview preparation advice.
- Do not recommend keyword stuffing, hidden text, false seniority, or unsupported metrics.
- Treat internal project names, customer names, repository names, and exact private metrics as approval-required.
- Maintain a strong one-page or two-page strategy based on seniority and evidence density, not arbitrary length.
